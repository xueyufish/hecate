## Context

P3 没有元智能体——系统维护依赖人工。P4 需要自动化系统维护的元智能体，定期扫描系统健康状态。

## Goals / Non-Goals

**Goals:**

1. 实现 GarbageCollectorAgent — 定期清理过期资源
2. 实现 ComplianceCheckerAgent — 定期扫描架构违规
3. 实现 DriftDetectorAgent — 检测配置漂移

**Non-Goals:**

1. 不实现自动修复（只报告，不自动修复）
2. 不实现强化学习优化（属于 p4-self-evolution）

## Decisions

### D1: 元智能体作为定时任务

**选择**：使用 Cron 调度元智能体执行

**理由**：
- 简单可靠
- 可配置执行频率
- 与现有定时任务框架集成

### D2: 只报告不修复

**选择**：元智能体只生成报告，不自动修复

**理由**：
- 自动修复可能引入新问题
- 人工审核更安全
- 渐进式：先报告，后自动修复

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 扫描频率过高影响性能 | 限制扫描范围和频率 |
| 误报过多 | 人工审核 + 反馈循环 |
| 扫描结果存储增长 | 设置 TTL 清理 |
